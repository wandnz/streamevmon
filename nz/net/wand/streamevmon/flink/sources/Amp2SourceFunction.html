    Sources are the beginning of a Flink pipeline.

    Sources are the beginning of a Flink pipeline. In our case, they produce Measurements to be passed to detectors. Measurements are obtained from outside this program.

    Sources can take two forms: SourceFunction and FileInputFormat. SourceFunctions allow gathering data with arbitrary methods, while a FileInputFormat is specifically for reading files, usually from disk.

Amp2SourceFunction 

class Amp2SourceFunction extends RichSourceFunction[Amp2Measurement] with HasFlinkConfig with CheckpointedFunction with Logging

Retrieves amp2 data from InfluxDB as a streaming source function. Supports fetching historical data (amount configured using fetchHistory), and then subscribing to new data, emitting that data as it arrives.

Configuration

See Influx connectors package object for configuration details. Any configuration given to overrideConfig from HasFlinkConfig will also be passed to the Influx connectors.
